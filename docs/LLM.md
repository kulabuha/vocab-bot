# Выбор и настройка LLM

Бот использует **OpenAI-совместимый** HTTP API (endpoint `POST /chat/completions`). Подойдёт любой провайдер с таким API или локальный сервер.

---

## Локальный LLM для тестирования (рекомендуется)

### Ollama

Удобно для разработки: один бинарник, модели качаются одной командой.

1. **Установка:** [ollama.com](https://ollama.com) — скачай и запусти. По умолчанию слушает `http://localhost:11434`, есть [OpenAI-совместимый endpoint](https://github.com/ollama/ollama/blob/main/docs/api.md#openai-compatible-api) по пути `/v1`.

2. **Какую модель взять** (бот генерирует коллокации и оценивает ответы в JSON — нужна модель, которая умеет следовать инструкциям и давать JSON):

   | Цель | Модель | Команда | Память ~ |
   |------|--------|---------|----------|
   | Быстрый тест, слабое железо | `phi3:mini` или `qwen2.5:0.5b` | `ollama pull phi3:mini` | ~2 GB |
   | Баланс скорость/качество | `llama3.2:3b` или `qwen2.5:3b` | `ollama pull llama3.2:3b` | ~2–4 GB |
   | Лучше качество, есть RAM | `mistral` или `llama3.1:8b` | `ollama pull mistral` | ~4–8 GB |

   Рекомендация для старта: **`llama3.2:3b`** или **`qwen2.5:3b`** — нормально держат JSON и английский, не слишком тяжёлые.

   **Потяжелее локально** (лучше следует инструкциям, стабильнее JSON и оценка ответов):
   | Модель | Команда | RAM (CPU) | С GPU |
   |--------|---------|-----------|--------|
   | **qwen2.5:7b** | `ollama pull qwen2.5:7b` | ~8–10 GB | 6 GB VRAM |
   | **llama3.1:8b** | `ollama pull llama3.1:8b` | ~8–10 GB | 6 GB VRAM |
   | **mistral** (7B) | `ollama pull mistral` | ~8 GB | 6 GB VRAM |
   | **llama3.1:70b** (если есть память) | `ollama pull llama3.1:70b` | 64+ GB / GPU 40+ GB | да |

   Для оценки ответов и генерации коллокаций **7B–8B** обычно хватает: меньше путает задание с ответом, лучше держит JSON. На Mac с 16 GB можно пробовать **qwen2.5:7b** или **mistral**; с 8 GB — оставаться на 3B или использовать облако.

   **Где взять полный список моделей (пожирнее и другие):**
   - **Официальный каталог Ollama:** [ollama.com/library](https://ollama.com/library) — все модели с размерами и описанием.
   - В терминале: `ollama list` — уже установленные; `ollama search <name>` — поиск по названию.

   **Пожирнее варианты** (чем больше — тем лучше инструкции и JSON, но нужна память):

   | Модель | Команда | RAM (примерно) | Когда ставить |
   |--------|---------|----------------|----------------|
   | **llama3.1:8b** | `ollama pull llama3.1:8b` | 8–10 GB | Уже в таблице выше |
   | **qwen2.5:14b** | `ollama pull qwen2.5:14b` | 12–16 GB | Сильнее 7B, хорошо держит формат |
   | **llama3.1:70b** | `ollama pull llama3.1:70b` | 48+ GB RAM или 40+ GB VRAM | Максимум качества локально |
   | **mistral-nemo:12b** | `ollama pull mistral-nemo:12b` | ~12 GB | Компактная 12B |
   | **deepseek-r1:14b** | `ollama pull deepseek-r1:14b` | ~16 GB | Рассуждения, сложные инструкции |

   После установки в `.env`: `LLM_MODEL=имя_модели` (например `LLM_MODEL=qwen2.5:14b`).

   ```bash
   ollama pull qwen2.5:7b
   # в .env: LLM_MODEL=qwen2.5:7b
   ```

3. **Запуск бота на хосте** (без Docker):
   ```bash
   export LLM_API_BASE=http://localhost:11434/v1
   export LLM_MODEL=llama3.2:3b
   export LLM_API_KEY=   # пусто для Ollama
   go run ./cmd/bot
   ```

4. **Бот в Docker, Ollama на хосте:**
   В `.env`:
   ```env
   LLM_API_BASE=http://host.docker.internal:11434/v1
   LLM_MODEL=llama3.2:3b
   LLM_API_KEY=
   ```
   На Windows/Mac `host.docker.internal` указывает на хост. На Linux добавь в `docker-compose.yml` для сервиса `bot`:
   ```yaml
   extra_hosts:
     - "host.docker.internal:host-gateway"
   ```

## Другие локальные варианты

| Решение      | Как подключать |
|-------------|-----------------|
| **LocalAI** | Поддерживает OpenAI API, можно подставить разные бэкенды (Ollama, llama.cpp и др.). Запуск в Docker, указать `LLM_API_BASE=http://localai:8080/v1`. |
| **LiteLLM** | Прокси к разным API (OpenAI, Anthropic, локальные). Один endpoint, в боте указываешь его как `LLM_API_BASE`. |
| **Open WebUI** | Если уже поднят с локальной моделью и OpenAI-совместимым API — укажи его в `LLM_API_BASE`. |

Во всех случаях задаёшь в `.env`: `LLM_API_BASE`, `LLM_MODEL`, при необходимости `LLM_API_KEY`.

---

## Облачные API

- **OpenAI:** `LLM_API_BASE=https://api.openai.com/v1`, `LLM_MODEL=gpt-4o-mini` (или `gpt-4o`), `LLM_API_KEY=sk-...`.
- **Другие провайдеры** (Azure OpenAI, Groq, Together, и т.д.): как правило, дают base URL и ключ — подставляешь в те же переменные. Модель — как указано в их доках.

---

## Переменные окружения (сводка)

| Переменная      | Обязательно | Описание |
|-----------------|------------|----------|
| `LLM_API_BASE`  | да         | Базовый URL API, например `http://localhost:11434/v1` или `https://api.openai.com/v1`. |
| `LLM_MODEL`     | нет         | Имя модели. По умолчанию в коде: `gpt-4o-mini`. Для Ollama: `llama3.2`, `phi3` и т.д. |
| `LLM_API_KEY`   | для облака  | Ключ API. Для Ollama можно оставить пустым. |
| `LLM_TIMEOUT_SEC` | нет       | Таймаут запроса в секундах (по умолчанию 60). |

Локально для тестов удобнее всего поднять **Ollama** и указать в `.env` её `LLM_API_BASE` и `LLM_MODEL`.

## Почему в ответах бота видно «\n» вместо переноса строки

В JSON строках перенос кодируется как **один** обратный слэш и буква n: `\n`. При разборе это даёт реальный символ новой строки. Если модель в значении `feedback` подставляет **два** символа — обратный слэш и букву n (как текст), то после парсинга в сообщении пользователю отображается именно «\n». Промпт просит модель использовать стандартный JSON-escape `\n` для переносов. Плюс в коде применяется нормализация: буквальные `\n` в ответе заменяются на реальные переносы (см. `internal/llm/normalize.go`). Если после обновления промпта и перезапуска переносы всё ещё не отображаются — значит модель продолжает выводить буквальные символы; нормализация должна их обработать.

Чтобы самому посмотреть, что именно возвращает модель, можно один раз вызвать градер и вывести сырой ответ и результат нормализации:

```bash
cd vocab-bot && go run ./cmd/llm-debug
```

(Нужны те же переменные окружения, что и для бота: `LLM_API_BASE`, `LLM_MODEL` и при необходимости `LLM_API_KEY`.)
